# NAICS-HS Matching Example

This is a sanitized example package for the `naics-hs-matching` skill. It uses synthetic clean-technology rows only. It does not contain S&P company-level data, real firm names, addresses, entity IDs, revenue, or headcount.

## What It Demonstrates

The sample mapping includes the main review cases:

- `candidate_match`: final NAICS appears in the candidate set.
- `manual_override`: final NAICS is outside the candidate set.
- `manual_only`: final NAICS exists but no candidate evidence exists.
- `unmapped_with_candidates`: candidates exist but no final NAICS was selected.
- `unmapped_no_candidates`: no final NAICS and no candidate evidence.
- Same HS code mapped to different NAICS across contexts.

## Files

| Path | Purpose |
|---|---|
| `input/synthetic_naics_hs_map.csv` | Starting synthetic mapping file. |
| `output/synthetic_naics_hs_map_with_review.csv` | Same file after review workflow fields are added. |
| `output/audit/` | Audit memo, row audit, issue files, and metrics generated by the skill script. |

## Reproduce the Example

Run from the root of `nzipl-claude-kit`:

```bash
python .claude/skills/naics-hs-matching/scripts/add_review_fields.py \
  examples/naics-hs-matching/input/synthetic_naics_hs_map.csv \
  examples/naics-hs-matching/output/synthetic_naics_hs_map_with_review.csv \
  --overwrite
```

Then audit it:

```bash
python .claude/skills/naics-hs-matching/scripts/audit_mapping.py \
  examples/naics-hs-matching/output/synthetic_naics_hs_map_with_review.csv \
  examples/naics-hs-matching/output/audit
```

## How To Read the Output

Start with:

- `output/audit/naics_hs_mapping_consistency_audit.md`
- `output/audit/issues_for_review.csv`
- `output/audit/manual_overrides.csv`
- `output/audit/unmapped_rows.csv`
- `output/audit/hs_multi_naics_review.csv`

The example is intentionally small, so the audit output can be inspected by hand.
